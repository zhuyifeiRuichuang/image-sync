# 2026-07-10 工作日志

## 修复 skopeo --verbose 参数不兼容 (commit 3760480)
- 问题：Ubuntu 默认 apt 源安装的 skopeo 版本不支持 `--verbose` 参数，导致所有 copy 命令失败
- 错误信息：`unknown flag: --verbose`
- 修复：移除 `scripts/sync-image.sh` 中所有 `skopeo copy` 命令的 `--verbose` 参数
- 影响位置：
  - manifest list 同步命令
  - 逐架构同步命令
- 与镜像本身无关，Docker Hub 上 apache/seatunnel:2.3.13 确实存在 amd64 和 arm64 架构

## 改用下载新版 skopeo binary (commit 7b97be7)
- 问题根因：Ubuntu apt 源的 skopeo 版本太旧（约 1.13.x），不支持 --verbose、--retry-times 等新参数
- 解决方案：改用 lework/skopeo-binary v1.23.0 预编译 binary
  ```bash
  curl -sL "https://github.com/lework/skopeo-binary/releases/download/v1.23.0/skopeo-linux-amd64" -o /usr/local/bin/skopeo
  chmod +x /usr/local/bin/skopeo
  ```
- 恢复了 --verbose 参数，新版完全支持
- 不用 docker 方案的原因：docker pull 只能拉取当前平台架构，多架构需逐架构 pull+manifest 合并，流程复杂且 manifest 是实验性功能

## 修复 registries.conf v2 格式兼容 (commit 36d529a)
- 问题：新版 skopeo v1.23.0 要求 `/etc/containers/registries.conf` 是 v2 格式，但 ubuntu-latest runner 默认是 v1 格式
- 错误信息：`registries.conf must be in v2 format but is in v1`
- 修复：在安装 skopeo 步骤中创建 v2 格式的 registries.conf
  ```bash
  echo 'unqualified-search-registries = ["docker.io"]' | sudo tee /etc/containers/registries.conf
  ```

## 彻底移除 --verbose 参数 (commit 80732b2)
- 查阅 skopeo 官方文档确认：skopeo copy **没有 --verbose 参数**
- skopeo 只有 `--quiet`（抑制输出）和全局 `--debug`（调试模式，放在子命令前）
- 彻底移除所有 skopeo copy 命令的 `--verbose` 参数
- run_with_heartbeat 已能捕获 skopeo 默认输出，无需额外 verbose

## 简化配置移除 REGISTRY_TYPE (commit 3c8e1b1)
- 移除 REGISTRY_TYPE，所有平台统一扁平命名空间
- 配置只需 4 项：REGISTRY_URL + REGISTRY_NAMESPACE (Variables) + REGISTRY_USERNAME + REGISTRY_PASSWORD (Secrets)
- 脚本从9参数简化为8参数
- 目标镜像格式：REGISTRY_URL/REGISTRY_NAMESPACE/镜像名最后一段:Tag
