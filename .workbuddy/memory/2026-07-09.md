## 增强错误提示详细信息 (commit 3038a6b)
- 脚本各阶段失败时输出 ERROR_TYPE 和 ERROR_DETAIL：
  - login_failed：捕获 skopeo login 的具体错误输出和退出码
  - source_image_not_found：捕获 skopeo inspect 的具体错误输出和退出码
  - sync_failed：捕获多架构和逐架构同步的退出码及详情
  - partial_sync_failed：区分部分架构失败
- Workflow 失败评论根据 ERROR_TYPE 分类展示：
  - login_failed：展示认证失败详情 + 针对性建议（密码类型说明、本地验证命令）
  - source_image_not_found：展示源镜像检查失败详情 + 建议检查拼写和私有镜像
  - sync_failed：展示同步失败详情 + 建议检查命名空间和权限
- 主流程改为逐阶段 if 检查，失败时立即终止并输出错误类型
- run_with_heartbeat 保存命令输出到全局变量 LAST_CMD_OUTPUT 供诊断

## 修复 issue 重复触发 Action (commit a0a69c0)
- 原因：创建 Issue 时模板自动带 image-sync 标签，会同时触发 opened 和 labeled 两个事件
- 修复：在 workflow 顶部添加 concurrency 配置，同一 issue 的并发运行自动取消旧的
  ```yaml
  concurrency:
    group: ${{ github.workflow }}-${{ github.event.issue.number }}
    cancel-in-progress: true
  ```
